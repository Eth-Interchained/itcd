Consensus from the inside out

Traditional Bitcoin nodes store whatever the network agrees on. NEDB makes causal order a storage invariant — the node cannot write a block that breaks causality without the storage layer rejecting it.

◈
Causal Provenance
Every block PUT includes caused_by: prev_block_hash. Blockchain causality is encoded at the storage layer — not just the application layer.
caused_by: prev_block_hash
⏱
AS OF Time-Travel
Any UTXO set at any historical block height — from inside the node itself. No archive node, no external service. AS OF seq works natively.
AS OF seq 500000
∑
Deterministic State Root
The BLAKE2b-512 chain head after writing a block's transactions is deterministic. Two nodes that processed the same chain produce identical heads — a storage-layer consensus proof.
GetStateRoot() → blake2b head
↻
MVCC Reorgs
Chain reorganisations become MVCC branches. The old chain state survives intact — no LevelDB delete-and-rewrite. Both forks coexist until the longest chain wins.
reorg = MVCC branch
🔐
AES-256-GCM Encryption
Phase 2: set NEDB_TMK and the full block index + chainstate is AES-256-GCM encrypted at rest. No LevelDB XOR obfuscation needed.
NEDB_TMK=<32-byte-hex>
◐
TRACE Provenance
Walk any UTXO's causal chain back to genesis from inside the node. TRACE caused_by REVERSE — no block explorer, no external API.
TRACE caused_by REVERSE
// leveldb vs nedb

Why the swap matters

LevelDB is a dumb key-value store — it has no concept of causality, history, or identity. NEDB makes all three native to the storage layer.

Capability LevelDB NEDB DAG
Causal structure ✗ none ✓ caused_by on every write
Historical UTXO state ✗ archive node required ✓ AS OF seq — any height, any node
State root ✗ not native to Bitcoin ✓ BLAKE2b-512 chain head per block
Chain reorgs Delete + rewrite UTXO entries ✓ MVCC branch — old state intact
At-rest encryption XOR obfuscation ✓ AES-256-GCM (Phase 2)
Tamper evidence ✗ trust the network ✓ verify() — cryptographic proof
Block explorer External service required ✓ TRACE caused_by inside the node
Consensus impact Storage is passive — receives consensus ✓ Storage enforces causal order
